# Fabrica - Technology Stack Specification

**Version:** 1.0  
**Last Updated:** November 19, 2025  
**Target Market:** Ethiopian Creator Economy

## Executive Summary

This document details the complete technology stack for Fabrica, with justifications for each technology choice based on Ethiopian market requirements, AI coding tool compatibility, cost-effectiveness, and scalability needs.

**Core Philosophy:** Use battle-tested, well-documented technologies with excellent AI tooling support to enable rapid development while maintaining enterprise-grade quality.

## 1. Frontend Stack

### 1.1 Framework: Next.js 14.2+

**Official Site:** https://nextjs.org  
**License:** MIT  
**Current Version:** 14.2.3  

**Why Next.js:**
1. **Server-Side Rendering (SSR):** Critical for low-bandwidth Ethiopian networks
   - Initial HTML rendered on server = faster perceived load times
   - SEO-friendly for marketing pages
2. **App Router:** Modern routing with layouts, loading states, error boundaries
3. **API Routes:** Full-stack framework (no separate backend needed for MVP)
4. **Image Optimization:** Automatic WebP conversion, responsive images
5. **Code Splitting:** Automatic, optimized bundle sizes
6. **AI Tool Support:** Excellent support in Cursor, Bolt.new, Lovable.dev
7. **Vercel Integration:** Zero-config deployment, edge functions
8. **Middleware:** Custom request handling (auth, rate limiting, A/B testing)

**Alternatives Considered:**
- **Remix:** Similar benefits but smaller ecosystem, less AI tooling
- **SvelteKit:** Smaller bundles but less mature, harder for AI tools
- **React SPA (Vite):** Worse initial load times on slow networks

**Configuration:**
```javascript
// next.config.js
/** @type {import('next').NextConfig} */

```

### 1.5 State Management: Zustand 4.5+

**Official Site:** https://github.com/pmndrs/zustand  
**License:** MIT  
**Bundle Size:** 1.2KB

**Why Zustand:**
1. **Lightweight:** 1KB vs Redux (10KB+)
2. **Simple API:** No boilerplate, easy to learn
3. **TypeScript:** Excellent type inference
4. **DevTools:** Redux DevTools compatible
5. **No Context:** Avoids React context re-render issues
6. **Persistence:** Built-in localStorage sync

**Example Store:**
```typescript
// stores/useCartStore.ts
import { create } from 'zustand'
import { persist } from 'zustand/middleware'

interface CartItem {
  productId: string
  title: string
  price: number
  quantity: number
}

interface CartStore {
  items: CartItem[]
  addItem: (item: CartItem) => void
  removeItem: (productId: string) => void
  clearCart: () => void
  total: number
}

export const useCartStore = create<CartStore>()(
  persist(
    (set, get) => ({
      items: [],
      addItem: (item) => set((state) => ({
        items: [...state.items, item]
      })),
      removeItem: (productId) => set((state) => ({
        items: state.items.filter(i => i.productId !== productId)
      })),
      clearCart: () => set({ items: [] }),
      get total() {
        return get().items.reduce((sum, item) => sum + item.price * item.quantity, 0)
      }
    }),
    {
      name: 'fabrica-cart'
    }
  )
)
```

### 1.6 Data Fetching: TanStack Query (React Query) 5.2+

**Official Site:** https://tanstack.com/query  
**License:** MIT

**Why React Query:**
1. **Caching:** Automatic request deduplication and caching
2. **Background Updates:** Stale-while-revalidate pattern
3. **Optimistic Updates:** Update UI before server confirms
4. **Retry Logic:** Automatic retries with exponential backoff
5. **Loading States:** Built-in loading, error, success states
6. **Devtools:** Excellent debugging experience

**Example Usage:**
```typescript
// hooks/useProducts.ts
import { useQuery } from '@tanstack/react-query'
import { supabase } from '@/lib/supabase/client'

export function useProducts(creatorId: string) {
  return useQuery({
    queryKey: ['products', creatorId],
    queryFn: async () => {
      const { data, error } = await supabase
        .from('products')
        .select('*')
        .eq('creator_id', creatorId)
        .eq('status', 'active')
      
      if (error) throw error
      return data
    },
    staleTime: 5 * 60 * 1000, // 5 minutes
    cacheTime: 10 * 60 * 1000, // 10 minutes
  })
}
```

### 1.7 Forms: React Hook Form + Zod

**React Hook Form:** https://react-hook-form.com  
**Zod:** https://zod.dev

**Why This Combo:**
1. **Performance:** Minimal re-renders (uncontrolled components)
2. **DX:** Simple API, great TypeScript support
3. **Validation:** Zod provides type-safe schema validation
4. **Small Size:** React Hook Form (9KB) + Zod (8KB)
5. **shadcn/ui Integration:** Built-in Form components

**Example:**
```typescript
// components/ProductForm.tsx
import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import * as z from 'zod'

const productSchema = z.object({
  title: z.string().min(3, 'Title must be at least 3 characters'),
  description: z.string().max(500, 'Description too long'),
  price: z.number().min(0.01, 'Price must be positive'),
  type: z.enum(['digital', 'booking', 'link']),
})

type ProductFormData = z.infer<typeof productSchema>

export function ProductForm() {
  const form = useForm<ProductFormData>({
    resolver: zodResolver(productSchema),
    defaultValues: {
      title: '',
      description: '',
      price: 0,
      type: 'digital',
    },
  })
  
  const onSubmit = async (data: ProductFormData) => {
    // Handle submission
  }
  
  return (
    <form onSubmit={form.handleSubmit(onSubmit)}>
      {/* Form fields */}
    </form>
  )
}
```

### 11.3 Rate Limiting: Upstash Redis

**Official Site:** https://upstash.com  
**Pricing:** Free tier (10K commands/day) ‚Üí $0.20/100K commands

**Why Upstash:**
1. **Serverless Redis:** No server management
2. **Edge-Compatible:** Works with Vercel Edge Functions
3. **Global Replication:** Multi-region
4. **Pay-per-request:** No idle charges
5. **Ethiopian Users:** Low latency (global edge)

**Implementation:**
```typescript
import { Ratelimit } from '@upstash/ratelimit'
import { Redis } from '@upstash/redis'

const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(100, '1 m'),
  analytics: true,
})

export async function POST(req: Request) {
  const ip = req.headers.get('x-forwarded-for') || 'anonymous'
  const { success } = await ratelimit.limit(ip)
  
  if (!success) {
    return new Response('Rate limit exceeded', { status: 429 })
  }
  
  // Process request
}
```

**Rate Limits by Endpoint:**
- Public API: 100 req/min per IP
- Authenticated API: 200 req/min per user
- Payment initiation: 10 req/hour per user
- Webhooks: 1000 req/min per source
